## 스크립트 작성 패턴
1. 기존 책 스크립트 (예: `import-bible-greek-content-thessalonians1.mjs`, `import-bible-greek-content-timothy1.mjs`) 의 구조를 그대로 따를 것.
2. `CONTENT` 객체에 장→절→`{kr, words, infos}` 의 3중 중첩.
3. `applyInfoToTokens()` 가 `infos` 의 헬라어 키를 자동으로 `greekTokens` 에 매핑.
4. **다이어크리틱(악센트) 무시 매칭**이므로 키는 원문 그대로 복사 권장.

## kr 작성 규칙
- 헬라어 어순을 한글 어미로 풀되, 토큰 단위로 `—`(em-dash) 로 구분.
- 의문, 명령, 부사절 등의 헬라어 어순을 유지하면서 한글 어조사를 붙임.

## words 작성 규칙
- 한 절당 1~3문장.
- 절의 핵심 주제 + 어휘 특이성(신약 본 절에만, 평행 구절 등) + 신학적/문학적 강조.
- **"결정적", "결정", "결정적 결정" 같은 어휘는 절제** (생성 모델이 반복 폭주에 빠지기 쉬움). 절 1개에 1회 이내로.

## infos 작성 규칙
- 한 절당 3~10개의 핵심 어휘.
- `"헬라어단어": "기본형 품사 (격/수/태/시제) — '한글뜻'. 부연(평행 구절 등)."` 형식.
- 분사·완료시제처럼 형태가 두드러진 단어를 우선 선정.
- 어원 (영어 어원, 신약 본 절에만 등) 을 짧게 부연.

## 실행 절차
```bash
# 1. 스크립트 작성
$EDITOR scripts/import-bible-greek-content-<book>.mjs

# 2. 임포터 실행
node scripts/import-bible-greek-content-<book>.mjs 2>&1 | tee /tmp/<book>-import.log

# 3. 매칭 안 됨 확인 (있으면 다이어크리틱/형태 불일치 → 키 수정 후 재실행)
grep "매칭 안 됨" /tmp/<book>-import.log

# 4. 100% 검증
node scripts/check-bible-greek-status.mjs <book>
```

## 주의사항
- `stripDiacritics()` 가 다이어크리틱·소문자·아포스트로피를 정규화하므로 키는 원문 그대로 두는 게 안전.
- 한 절에 `info` 토큰이 너무 많으면 (10개 초과) UI 가 복잡해짐 — 3~6 개 권장.
- 절 분할이 SBLGNT 와 개역한글에서 다를 수 있음 — `greekKr` 의 절 번호는 한글 절 번호를 따름.
