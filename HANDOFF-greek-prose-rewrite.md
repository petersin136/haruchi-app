# 하루치 앱 — "헬라어" 모드 본문 의역체 재작성 인계

## 문제 인식

- `app/bible-reading/<book>.json` 의 `greekKr` 는 본래 **로마서(완료) 같은 자연스러운 한국어 의역체**여야 함
  - 예: 로마서 1:1 = "예수 그리스도의 종 바울 — 사도로 부르심을 받아 하나님의 복음을 위해 따로 세움을 받은 사람입니다."
- 그러나 어젯밤 자동생성기로 만든 신약 26권은 모두 **"단어 — 단어 — 단어" 단어 단위 기계적 포맷**이라 의역이 전혀 안 되어 있음
  - 예: 요 1:1 (수정 전) = "하나 — 시작 — ~이었다 — 그(는) — 말씀 — 그리고 — ..."
- 즉 `greekKr` 만 보고 절의 의미가 자연스럽게 읽혀야 하는데, 현재는 단어 나열만 있는 상태
- `greekWords` 와 `infos` 는 (대체로) 사용 가능한 수준 — `greekKr` 만 재작성 필요

## 책별 상태

| 상태 | 책 |
|------|---|
| ✅ 의역체 완료 | 로마서 (433절) |
| 🔧 일부 진행 중 | 요한복음 — 1–2장(76절) 의역체 완료 |
| ❌ 기계적 단어 단위 | 마태·마가·누가·요한(3–21장)·사도행전·고전·고후·갈·엡·빌·골·살전·살후·딤전·딤후·딛·몬·히·약·벧전·벧후·요일·요이·요삼·유·계 |

## 작업 방식

1. **재작성 대상은 `greekKr` 만**. `greek`(원문), `greekTokens`(이미 info 부여됨), `greekWords`(절 풀이) 는 보존.
2. 작성 스타일: 로마서를 그대로 따른다.
   - **1–3 문장의 자연스러운 한국어 의역**
   - 직접 인용은 `— "..."` 또는 `— "..."` 로 처리
   - em-dash 는 한 문장 안에서 강조나 해석 삽입용 1–2회 정도까지 허용 (기계적 단어 구분에는 사용 금지)
3. 챕터별 작업 스크립트 위치: `scripts/import-john-chapter.mjs` (CONTENT 객체에 챕터별 데이터 누적)
4. 실행: `node scripts/import-john-chapter.mjs` — 이미 완료된 절도 덮어쓰므로 안전
5. 검증: 
   ```bash
   node -e "const d=JSON.parse(require('fs').readFileSync('app/bible-reading/john.json','utf8')); let m=0,t=0; for(const ch of d.chapters){for(const v of (ch.verses.greekKr??[])){t++; const dashes=(v.t.match(/—/g)||[]).length; if(dashes>=5 && (dashes*15)/v.t.length > 1) m++;}} console.log('mechanical:'+m+'/'+t);"
   ```

## 다음 단계 (요한복음 우선)

요한복음 3장(36절) — 22장(25절)까지 = **828절**.
한 세션 당 1–2 챕터(50–80절) 처리 가능.

### 챕터 단위 우선순위
1. 요 3장 (36절) — 니고데모, 거듭남
2. 요 4장 (54절) — 사마리아 여인
3. 요 5장 (47절) — 베데스다, 아버지·아들 담화
4. 요 6장 (71절) — 오병이어, 생명의 떡
5. 요 7–10장 (장막절·맹인·선한 목자)
6. 요 11장 — 나사로
7. 요 12–17장 — 고별담화
8. 요 18–21장 — 수난·부활·회복

### 그 다음 우선순위 (요한복음 끝나면)
공관복음 → 사도행전 → 고전·고후 → 나머지 서신서·요한계시록

## 사용 자료

- `scripts/_greek-lexicon.json`, `scripts/_lemma-lexicon.json`, `scripts/_common-lex.json` — info 토큰용 사전 (이미 적용됨)
- `app/bible-reading/<book>.json` — 데이터 파일
- 로마서·요한복음 1–2장 — 의역체 스타일 참조

## 작업자 메모

- 한 응답당 약 20–25 절이 안정적인 한계 (StrReplace 호출 횟수 + 출력 토큰)
- 같은 챕터를 두 응답에 걸쳐 끝내기보다, 한 응답 안에 한 챕터를 다 끝내는 것이 일관성 측면에서 유리
- info 토큰은 이미 자동생성기로 들어가 있으므로 추가/수정은 선택사항. 우선은 `greekKr` 의역체만 집중
